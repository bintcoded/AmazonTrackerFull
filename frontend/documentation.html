Project Overview

The Amazon Endangered Species Tracker is a web-based relational database application developed as part of a university database course at Wayne State University. The application allows users to explore, manage, and query information about endangered and threatened wildlife species in the Amazon rainforest ecosystem.

The database tracks species across 4 organism types (Mammals, Birds, Reptiles, and Amphibians), covering both currently threatened and historically extinct species. Data is sourced from the IUCN Red List, BirdLife International, the WWF Living Planet Report 2024, and peer-reviewed academic literature including Amazonian Mammals: Current Knowledge and Conservation Priorities (Springer, 2023).

The application demonstrates a normalized relational database design using 6 interconnected tables with proper primary and foreign key constraints, a many-to-many junction table (speciesthreats), and SQL queries covering SELECT, INSERT, UPDATE, and DELETE operations.

Setup Instructions

Step 1: Install PostgreSQL

Download and install PostgreSQL 18+ from postgresql.org/download. During setup, note the superuser password you create. Also install pgAdmin 4 for a visual interface.

Step 2: Create the Database

Open pgAdmin, connect to your local server, and create a new database named Amazon Species. Alternatively, run this in psql:

CREATE DATABASE "Amazon Species"
  WITH TEMPLATE = template0
  ENCODING = 'UTF8'
  LOCALE = 'English_United States.1252';

Step 3: Restore the Database from Backup

Use the provided Amazon_Species_Database_.backup file to restore all tables and data:

# Using pg_restore on the command line:
pg_restore -U postgres -d "Amazon Species" Amazon_Species_Database_.backup

# Or in pgAdmin:
# Right-click the database → Restore → select the .backup file

Step 4: Verify the Tables

After restoring, confirm all 6 tables exist and contain data:

-- Run in pgAdmin Query Tool:
SELECT table_name FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;

-- Expected tables:
-- organismtype, region, species, speciesthreats, status, threat

Step 5: Run the Web Interface

Navigate to the project folder and start a local server:

# Option 1 — Python (Python 3):
cd AmazonTracker/
python -m http.server 8080
# Then open: http://localhost:8080/index.html

# Option 2 — VS Code Live Server:
# Open the AmazonTracker/ folder in VS Code
# Right-click index.html → Open with Live Server

Database Schema

The database uses PostgreSQL with 6 tables in the public schema:

TablePrimary KeyForeign KeysDescription
speciesspeciesidorganismtypeid, statusid, regionidCore table, one row per species
organismtypeorganismtypeidNoneAnimal classification (Mammal, Bird, etc.)
statusstatusidNoneIUCN conservation status with risklevel
regionregionidNoneAmazon sub-regions and ecosystems
threatthreatidNoneEnvironmental threat categories
speciesthreats(speciesid, threatid)speciesid, threatidMany-to-many junction table

Key SQL: Create Tables

CREATE TABLE public.organismtype (
    organismtypeid integer NOT NULL,
    type           character varying(100),
    organismdescription character varying(300),
    CONSTRAINT organismtype_pkey PRIMARY KEY (organismtypeid)
);

CREATE TABLE public.status (
    statusid   integer NOT NULL,
    statusvalue character varying(200),
    risklevel  integer,
    CONSTRAINT status_pkey PRIMARY KEY (statusid)
);

CREATE TABLE public.region (
    regionid   integer NOT NULL,
    regionname character varying(200),
    ecosystem  character varying(200),
    CONSTRAINT region_pkey PRIMARY KEY (regionid)
);

CREATE TABLE public.threat (
    threatid  integer NOT NULL,
    threatname character varying(200),
    CONSTRAINT threat_pkey PRIMARY KEY (threatid)
);

CREATE TABLE public.species (
    speciesid      integer NOT NULL,
    commonname     character varying(200),
    scientificname character varying(200),
    organismtypeid integer REFERENCES public.organismtype(organismtypeid),
    statusid       integer REFERENCES public.status(statusid),
    regionid       integer REFERENCES public.region(regionid),
    habitat        character varying(200),
    CONSTRAINT species_pkey PRIMARY KEY (speciesid)
);

CREATE TABLE public.speciesthreats (
    speciesid integer REFERENCES public.species(speciesid),
    threatid  integer REFERENCES public.threat(threatid),
    CONSTRAINT speciesthreats_pkey PRIMARY KEY (speciesid, threatid)
);

Example SQL Queries:

-- List all species ordered by extinction risk
SELECT s.commonname, s.scientificname,
       st.statusvalue, st.risklevel,
       o.type AS organism_type,
       r.regionname
FROM species s
JOIN status st ON s.statusid = st.statusid
JOIN organismtype o ON s.organismtypeid = o.organismtypeid
JOIN region r ON s.regionid = r.regionid
ORDER BY st.risklevel ASC, s.commonname ASC;

-- Find all threats for a specific species
SELECT s.commonname, t.threatname
FROM speciesthreats st
JOIN species s ON st.speciesid = s.speciesid
JOIN threat t ON st.threatid = t.threatid
WHERE s.speciesid = 1;

User Guide

A. Home Page (index.html)

The main landing page introduces the Amazon Endangered Species Tracker project. It displays key statistics about the database, background information on the Amazon rainforest, the data sources used, and navigation cards linking to each section of the application. No interaction is required. Simply click any card or the navigation menu to explore.

B. Add Species (add.html)

This page contains a form to insert a new species into the database. Fill in the common name, scientific name, organism type, IUCN conservation status, region, and primary threat. The SQL Preview sidebar updates in real time to show the exact INSERT statement that would be sent to the database. Click Insert Record to submit. Required fields are marked with a red asterisk (*).

C. Delete Species (delete.html)

Search for a species by name or ID using the search box. Matching records appear as clickable results. Selecting a record shows a full record card with a confirmation step before deletion. The SQL preview shows both the child-record DELETE (from speciesthreats) and the parent-record DELETE (from species), because foreign key constraints require child records to be removed first.

D. Update Species (edit.html)

Search for a species by name or ID. Selecting a result populates all fields with the current values. Edit any field: common name, scientific name, organism type, status, region, population, or habitat. Click Save Changes. The SQL Preview sidebar shows the UPDATE statement with the current field values in real time.

E. View Species (view.html)

Browse all species in the database using the search bar and filter controls. Filter by IUCN risk level (using the colored legend buttons), animal type, or primary threat. Use the sort dropdown to order by extinction risk, name A to Z, or animal type. Each species card shows the scientific name, status badge, region, population estimate, and a brief description.

F. Documentation (documentation.html)

This page. Contains the project overview, team information, software and hardware requirements, step-by-step setup instructions, full database schema with CREATE TABLE statements, example SQL queries, and the data sources bibliography.
